# Security Policy

## Supported Versions

Este repositório usa governança centralizada. Incidentes de segurança devem ser
reportados imediatamente, independentemente da versão afetada.

## Reporting a Vulnerability

Não abra issue pública para vulnerabilidades.

Use um dos canais abaixo:

- Email: `security@isis-saude.example`
- Canal interno: `#governance-incidents`

Inclua:

- impacto e severidade estimada
- repositório(s) afetado(s)
- passos de reprodução
- evidências (logs, hashes, screenshots)

## SLA interno

- triagem inicial em até 24h
- plano de mitigação em até 72h para casos críticos
